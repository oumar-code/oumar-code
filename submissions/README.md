# Submissions

This folder contains judging-ready submission assets and packaging workflow.

---

## Packaging Workflow

```
submissions/
├── README.md               ← this file
└── assets/
    ├── demo-script-2min.md               ← live 2-min pitch script
    ├── demo-script-backup.md             ← backup if live demo fails
    ├── technical-writeup.md              ← fill before submission
    ├── architecture-diagram-instructions.md  ← how to export diagram
    ├── benchmark-table.md                ← before/after optimisation table
    ├── impact-narrative.md               ← who/why/why-us template
    └── final-day-runbook.md              ← step-by-step submission day guide
```

## Artifact Checklist (complete before submitting)

### Core Assets
- [ ] `assets/technical-writeup.md` — filled and proofread
- [ ] `assets/benchmark-table.md` — real numbers, not stub values
- [ ] `assets/impact-narrative.md` — concise, judge-friendly
- [ ] `assets/architecture-diagram.png` — exported (see instructions)
- [ ] `benchmarks/report.md` — generated by `make benchmark`
- [ ] `benchmarks/report.json` — machine-readable version

### Per-Challenge
- [ ] `challenges/arm-create/submission-checklist.md` — all boxes checked
- [ ] `challenges/backblaze-genblaze/submission-checklist.md` — all boxes checked
- [ ] `challenges/qwen-cloud/submission-checklist.md` — all boxes checked

### Demo
- [ ] 2-min demo rehearsed (≥ 3 times)
- [ ] Backup demo script printed or on phone

## How to Switch Challenges in Under 5 Minutes

1. Pick challenge name: `arm-create` | `backblaze-genblaze` | `qwen-cloud`
2. Check KPIs: open `challenges/<name>/kpi.md`
3. Review config: `challenges/<name>/config.safe.env`
4. Run: `make eval CHALLENGE=<name> PROFILE=safe`
5. Review output in `benchmarks/`

That's it. The shared core never changes.
